📌 길이, 크기를 표현하는 다양한 단위 ​ 웹 사이트에서 길이, 크기를 표현하는 속성은 굉장히 많습니다. 예를 들어, width/height 같이 너비, 높이를 직접적으로 설정하기도 하고, 폰트 크기(font-size), 여백(padding, margin) 을 설정하기도 합니다. ​ 이렇게 크기, 길이를 표현할 때 사용할 수 있는 단위에는 여러가지 종류가 있습니다. 그 중에서 가장 중요하고, 자주 사용되는 4가지를 알아보도록 하겠습니다. ​ ​ ✅ 1. 픽셀(px) 아마 가장 많이 보았을 단위입니다. 해상도를 표현할 때 픽셀(px) 단위를 많이 사용하는데, 그 때 사용하는 단위와 같습니다. 변하지 않는 절대값입니다. ​ ​ ✅ 2. em / rem em과 rem은 상대값 입니다. 모든 html 태그는 중첩이 가능하다고 했었죠~ em은 상위(부모)요소의 글자 크기(font-size)를 1em으로 두고 계산하며, rem은 최상위(html 태그) 요소의 글자 크기(font-size)를 1rem으로 두고 계산합니다. ​ 말이 조금 어렵죠..?😥 예를 들어볼게요. ​ em은 상위 요소의 글자 크기를 1em으로 둡니다. 따라서 마지막 width: 2em의 상위 요소 div의 글자 크기는 20px로 설정되어 있죠. 따라서 2em은 20px * 2 = 40px로 계산됩니다. ​ rem은 최상위 요소의 글자 크기를 1rem으로 둡니다. 따라서 마지막 width: 2rem의 최상위 요소 html 태그의 글자 크기가 10px이므로 2rem은 20px로 표시됩니다. ​ 그럼 이렇게 복잡한... em과 rem은 도대체 왜? 언제? 누가? 사용할까요? ​ 상황에 따라 px, em, rem 중 선택하여 사용하게 됩니다. 반응형 웹사이트를 만들 경우 px보다는 em, rem을 권장합니다. (반응형 웹사이트에 대해서는 아래에 설명 해놓았습니다 🙂) ​ "화면 크기가 변할 때 변해야 하는 수치인가? = em, rem / 화면 크기가 변해도 변하지 않는 수치인가? = px" 이러한 기준으로 px과 em/rem 중에 선택해서 사용하면 됩니다. ​ em과 rem 중에서는 개인 별로 선호도 차이가 있을 수 있으나 em의 경우 부모 요소에 영향을 받으므로 헷갈릴 확률이 높아서 주로 rem을 많이 사용하게 되는 것 같습니다 :) (정답은 없어요~) ​ ​ ✅ 3. %(퍼센트) 마지막으로 설명할 단위는 퍼센트(%)입니다. 일반적으로도 많이 사용하는 % 단위의 의미처럼 전체 크기 중에 비율로 설정할 수 있습니다. ​ 폰트 크기보다는 너비나 높이 등에서 주로 사용됩니다. 주의해야 할 점은, 퍼센트에서의 비율도 상위 요소 기준이라는 것 입니다. ​ ​ 설명한 px, em, rem, % 이외에도 vw, vh, vmin, vmax, ex, ch 등이 있습니다. 경우에 따라 필요한 단위를 사용한다는 점, 주로 px, em, rem, % 가 사용된다는 점만 기억해주세요! ​ ​ 📌 [심화] 반응형 웹 (Responsive Web) 이란? ​ 분명 같은 주소로 접속했는데 큰~~~~ 모니터로 볼 때, 12인치 노트북으로 볼때, 태블릿으로 볼 때, 핸드폰으로 들어갈 때 같은 디자인이지만 미묘하게 다른 사이트를 본 적이 있나요? ​ 이런 사이트들을 반응형 웹사이트라고 합니다. 반응형 웹은 하나의 웹 사이트가 PC, 스마트폰, 태블릿 등 다양한 사이즈의 화면(디스플레이)에 따라 자동으로 화면 크기가 변하도록 만드는 것​을 의미합니다. ​ 반응형 웹이 화면 크기에 따라 어떻게 바뀌는지 개발자 모드에서도 확인할 수 있습니다. ​ 아래의 이미지처럼 개발자 모드에서 요소 선택하는 버튼 옆의 버튼(빨간색 네모로 표시)을 누르면 왼쪽에 뜨는 화면이 바뀝니다. (Device Toolbar가 표시됩니다.) ​ 원하는 기기 종류를 선택하여 해당 기기에서 어떻게 보여지는지 확인할 수 있습니다. (Responsive를 선택하면 자유자재로 크기를 변경하며 변화를 확인할 수 있습니다.) ​ 여러분이 자주 들어가는 웹사이트를 들어가, 반응형으로 만들어있는지, 만들어져있다면 어떻게 바뀌는지 확인해보세요. ​ *주의* - 간혹 모바일 기기에서 주소가 바뀌는 사이트가 있습니다. (예를 들어 naver는 모바일로 접근하면 m.naver.com으로 바뀝니다.) - 이 경우 모바일 용 웹사이트를 별개로 디자인/개발한 사례이므로 반응형이라고 하기 어렵습니다. - 반응형 웹사이트의 경우 주소가 바뀌지 않고 화면 크기에 따라 디자인만 조금씩 변경됩니다. (예: 화면이 클 땐 사진첩에 사진이 한 줄에 6장이였다가 화면이 줄어들면 사진이 한 줄에 3장씩 있게 되는 경우) ​ ​ 반응형 웹사이트를 만들 때는 고려할 것이 많습니다. 크게 매우 큰 모니터(xl), 노트북 사이즈의 모니터(lg), 태블릿 사이즈(md), 모바일 사이즈(sm) 정도로 나누어 각 경우에 따라 디자인이 어떻게 변경될지 모두 고려해야 합니다. ​ 그렇기 때문에 반응형 웹에는 px보다 em, rem이 주로 사용됩니다. 화면 크기가 달라지면 자연스럽게 기본 글자 크기(html 태그의 font-size)가 변경되고 em, rem으로 크기를 설정할 경우 기본 글자 크기가 변경되면 자연스럽게 전체 크기도 변경되기 때문입니다. ​ *잘 이해가 되지 않는 다면, 스마트폰에서 기기 자체 글자 크기를 바꿨을 때 모든 앱의 글자 크기가 변경되는 것을 떠올리면 됩니다. ​ 따라서 html, css 웹 기초 문법이 어느정도 익숙해진 이후에 반응형 웹사이트 개발에 도전하는 것을 추천합니다. ⭐ 정리하기 ⭐ ✔ 웹 개발에서 (주로) 사용되는 단위는 px, em, rem, % - px: 절대 값! - em: 상위 요소의 폰트 크기를 기준으로 하는 상대값 - rem: 최상위 요소의 폰트 크기를 기준으로 하는 상대값 - %: 상위 요소를 기준으로 하는 비율값 ✔ (심화) 반응형 웹 - 화면 크기에 따라 변하는 웹사이트 - 반응형 웹사이트를 도전하기 전에 기초 html, css를 마스터하자. ​ ​ 콘텐츠가 도움이 되었다면 좋아요💗 댓글은 콘텐츠 제작에 큰 힘이 됩니다~
